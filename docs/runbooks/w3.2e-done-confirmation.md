## Notes on the subdomain-per-edition risk (plan for 2027)

The AAP seed URL pins the 2026 edition via the `am2026.perio.org` subdomain. When AAP launches `am2027.perio.org` (~Q1 2027):

1. Operator updates `config/sources.yaml`'s `aap_annual_meeting` seed_urls + homepage_url to `am2027.perio.org`.
2. Operator bumps the year string in `parsers/aap.py` from `2026` → `2027` (the hardcoded classifier tripwire per spec D7).
3. Re-run `seed-sources` + `run --source aap_annual_meeting`. New edition ingests as a new events row (different `starts_on`).

Current 2026 event row stays in the DB (historical record), just gets fewer future re-crawls. When the site tears down `am2026.perio.org`, the ingest will start returning 404s → parser emits `source_blocked` review_items. That's the signal to either de-activate the source or adjust `sources.yaml`.

## Residual risks (non-blocking)

- **Session-level extraction** still out of scope. AAP's schedule has 80+ individual sessions that could be separate event rows; MVP model is one events row per conference edition. Revisit in W4+ if operators want session granularity.
- **Multi-hotel venue model** not captured. Parser reports only the convention center as `venue_name`; the four "Official AAP Hotels" aren't stored. Not a data loss — the hotel page (`housing.html`) is still ingested as a source_pages entry, so content-hash changes would trigger re-processing. Future: a dedicated `venues` or `accommodations` table if operators want that surface.
- **Stale template anchors** like `#getting-to-toronto` still linger in AAP's HTML (noted in prep §6.2). Parser ignores all anchors, so no extraction impact. Operator visibility only.
