## Plan deviations (documented for audit)

- **`from uuid import UUID` added to `repositories/sources.py`.** The plan assumed this import already existed; it didn't. Added as part of Task 2.
- **Signature smoke test rewritten to signature inspection.** The plan's original test called `run_source(MagicMock(), source_code="does-not-exist", force=True)` and expected `ValueError`, but `get_source_by_code(MagicMock(), ...)` returns another MagicMock (truthy), so the None-branch didn't fire. Rewrote using `inspect.signature` + `typing.get_type_hints` to introspect the signature directly ‚Äî simpler and faithful to the spec's intent (¬ß4 D6 locks shape, not runtime behavior).

Both deviations are captured in the Task 5 commit message on `feat/w3-2a-source-run-bookkeeping`.

## Residual risks (non-blocking; carry to W3.2b+)

- `--force` has no behavioral effect yet. Threading is locked by the signature smoke test, but the first time the flag will actually do something is when W3.2b adds due-selection ‚Äî the test ensures W3.2b can wire it without refactoring the signature.
- Error bookkeeping uses `str(exc)` directly. Pathologically long or multi-line messages may surface awkwardly in the admin UI's red error callout. Not blocking W3.2b; address when operators report an issue.
- The fresh-session error write on source-not-found only succeeds if the source row exists. If the source row truly doesn't exist (which triggered the not-found in the first place), `_record_error_bookkeeping_fresh_session` silently returns without a write. This is by design ‚Äî there's nothing to update ‚Äî but means a persistently-misconfigured CLI invocation leaves no audit trail at the DB level. The pipeline still raises `ValueError`, which Typer will log; that's the audit trail for this case.
