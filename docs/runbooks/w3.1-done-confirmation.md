## Residual risks (carried forward from plan; non-blocking)

- **Markup drift on `h1.swiper-title`.** The detail classifier's first structural signal is fixture-backed; if GNYDM restructures the homepage hero, the detail extractor will return 0 events and no canary fails. Treat as normal parser maintenance (refresh fixture + re-tune classifier + re-run this runbook), not an incident.
- **`/images/logo-YYYY.png` pattern change.** If the site stops naming the logo asset with an embedded year (e.g. cache-busting hash suffix, different path), `_extract_year_from_logo` returns `None` → zero events. `test_homepage_without_logo_yields_zero_events` locks this behavior; again, treat as normal maintenance.
- **Partial-page content changes.** `pipeline._diff_event_fields` treats `None` in the incoming candidate as a deliberate clear. If one seed page's hash changes and the other's doesn't, a re-fetch of the changed page can clobber precedence-won fields from the unchanged page. Out of scope for W3.1; revisit in W3.2+ if precedence drift is observed in the wild.
- **Pre-existing mypy errors in `tests/` files.** `uv run mypy .` surfaces 11 pre-existing errors in `tests/test_ada_parser.py` and related test harness files; these existed on `main` before Phase 1 and are not introduced by W3.1. CI does not check these (CI mypy scope is `medevents_ingest` only). Suggested follow-up outside W3.1: either add type hints to the affected test harness functions or add `# type: ignore` suppressions.

## Deviations from the plan (documented for audit)

- **`_parse_homepage` extraction strategy.** The plan's Phase 2 Task 7 used `soup.get_text("\n", strip=True)` + `startswith("Meeting Dates:")` to locate the date line. The real homepage wraps ordinal suffixes in `<sup>th/st</sup>` tags that split across `\n`-separated lines, truncating `raw_date` to `"November 27"` instead of `"November 27th - December 1st"`. The implementer (Phase 2 commit `ec608bb`) located the `Meeting Dates:` text node directly via `soup.find(string=...)` and called `.parent.get_text("")` (empty separator) so the superscripts fold inline. Inline comment in `_parse_homepage` documents the choice. All 6 detail-classifier tests pass with this approach.
